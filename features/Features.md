# mealoptim — Features & Roadmap

A living document tracking what's shipped, what's in progress, and what's coming next.

---

## ✅ Shipped

### Core Experience
- **AI price comparison** — Claude analyzes any ingredient list and returns realistic prices per store
- **Mock data fallback** — app works fully offline or without an API key using a built-in data generator
- **4-step analyzing screen** — animated progress (parse → find stores → compare prices → optimize route)
- **3 optimization modes** — Best Price, Best Balance, Fastest Trip (switchable on both home and results screens)

### Recipe Input
- **Free-text ingredient paste** — one ingredient per line, any format
- **Optional recipe name** — shown in results header
- **4 example recipes** — Carbonara, Stir Fry, Avocado Toast, Margherita Pizza (one-tap fill)

### Results Screen
- **Store cards** — ranked by chosen mode, with distance, walk time, and drive time
- **BEST badge** — highlights the top-ranked store
- **Savings summary** — shows how much you save vs the priciest nearby option, in local currency
- **Ingredient breakdown** — per-item prices at the best store, with running total
- **AI insight** — one-sentence recommendation generated by Claude

### Map & Routing
- **SVG route map** — visual overhead view with store pins, distance rings, and a dashed route line
- **Store switcher** — tap any store to re-route without going back
- **Walk / Drive panels** — time and distance for each transport mode
- **Google Maps deep-link** — opens directions to the selected store in one tap

### Localization
- **20 countries** — GR, US, UK, DE, FR, ES, IT, AU, CA, JP, NL, SE, BR, IN, SG, CN, MX, ZA, PL, PT
- **Local supermarket chains** — 4–5 per country (e.g. Sklavenitis, Walmart, Tesco, Rewe, AEON…)
- **Local currencies** — €, $, £, ¥, kr, R$, ₹, S$, MX$, R, zł, A$, C$
- **Searchable country picker** — bottom-sheet modal with live filter

### Design
- **Dark theme** — near-black background (#0A0A08) with lime (#C4FF3C) accent
- **3 typefaces** — Syne (headings), DM Mono (data), Outfit (body)
- **Animated entry** — staggered fadeUp on all key elements
- **Hover states** — cards, store chips, example buttons

## 🗓 Roadmap

### v1.1 — Save & Remember
- [ ] **Saved recipes** — store favorite ingredient lists in localStorage
- [ ] **Search history** — last 5 scans, tappable to re-run
- [ ] **Shopping list export** — generate a plain-text or PDF checklist sorted by store aisle

### v1.2 — Smarter Budgeting
- [ ] **Budget mode** — set a max spend, Claude suggests substitutions to stay under it
- [ ] **Serving size scaler** — adjust ingredient quantities for 1–12 people, prices update live
- [ ] **Price trend indicator** — flag ingredients that are currently seasonal / expensive
- [ ] **Cheapest substitution suggestions** — "swap guanciale → pancetta to save €1.20"

### v1.3 — Meal Planning
- [ ] **Weekly meal planner** — add multiple recipes, consolidate into one master shopping list
- [ ] **Pantry tracker** — mark items you already have, deduct them from the list
- [ ] **Nutrition summary** — calories, macros, and allergens via an open food API
- [ ] **Dietary filters** — vegan, vegetarian, gluten-free, halal, kosher

### v1.4 — Community & Social
- [ ] **Recipe library** — browse and import crowd-sourced recipes from other users
- [ ] **Price crowd-sourcing** — users report actual shelf prices to improve accuracy
- [ ] **Referral / invite** — share a recipe scan with a friend, they open it pre-filled

### v1.5 — Store Integrations
- [ ] **Click & collect** — deep-link directly to a store's online basket with items pre-added
- [ ] **Loyalty card discounts** — factor in Tesco Clubcard, Sklavenitis card, etc.
- [ ] **Live stock checker** — warn if an item is out of stock at the chosen store
- [ ] **Delivery option** — show delivery fee and compare total cost including shipping

### v2.0 — Platform Expansion
- [ ] **PWA** — installable on mobile home screen, works offline with cached prices
- [ ] **iOS / Android app** — React Native port with native GPS and camera
- [ ] **Browser extension** — detect recipe pages automatically and trigger a scan
- [ ] **Grocery API integrations** — real-time prices from Instacart, Ocado, Skroutz

## 💡 Exploratory / Long-Term

Ideas worth validating before committing engineering time.

| Idea | Why it's interesting | Risk |
|---|---|---|
| **Carbon footprint score** | Show the environmental cost of each shopping choice | Data availability |
| **Price alert subscriptions** | Notify when a key ingredient drops below a threshold | Needs backend + auth |
| **Restaurant vs cook comparison** | "This pasta costs €3.20 to make vs €14 at a restaurant" | Claude can estimate |
| **Multi-store route** | Split the list across 2 stores if it saves enough | UX complexity |
| **Supermarket loyalty programme advisor** | Recommend which loyalty card to use at checkout | Region-specific |
| **Recipe cost tracking over time** | Has carbonara gotten more expensive this year? | Needs price history DB |

## 🐛 Known Issues

| Issue | Priority | Notes |
|---|---|---|
| API key exposed in browser | High | Needs a backend proxy for production |
| Map pins use fixed angles, not real geo | Medium | SVG map is illustrative, not geographic |
| No error screen if both API and mock fail | Medium | Add a graceful error state |
| Country picker doesn't persist on refresh | Low | Could use localStorage |
| Mobile keyboard pushes CTA button off screen | Low | Needs scroll-to-button on focus |
